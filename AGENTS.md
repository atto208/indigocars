# Indigo Cars website

Marketing site + admin CMS for a car rental company in Istanbul. NOT a booking engine â€” public "Book Now" buttons open WhatsApp (`wa.me`) with a pre-filled message.

## Architecture

- Next.js 16 App Router, Tailwind 4 (`@theme` tokens in `app/globals.css`), Prisma + SQLite (`prisma/dev.db`).
- Public site is **motion-driven** (`motion` / Framer Motion). Reusable interaction primitives live in `components/interactions.tsx`: `Reveal` (scroll fade-up), `Magnetic`, `TiltCard`, `Counter` (animated numbers), `Cursor` (custom follower, `[data-cursor]` enlarges it). The hero (`components/Hero.tsx`) is a full-bleed cinematic **video** (`public/video/hero.mp4`, poster `hero-poster.jpg`) with a kinetic word-stagger headline; video was generated with Higgsfield Veo3 from a nano-banana still. Fleet cards tilt + filter with animated layout; `components/Sections.tsx` has the bento "Why Us" (`#why`), About, Contact, Footer. Keep the body light; the hero is brand-indigo cinematic (the owner asked for "special/interactive", that's why the hero is video â€” don't flatten it back to a plain header).
- All editable site text/photos live in the `Setting` key-value table; defaults + keys in `lib/settings.ts` (`DEFAULT_SETTINGS`). The content form in `app/admin/content/page.tsx` iterates those keys.
- `Car` = one row per model shown on the website (deduplicated). `FleetCar` = physical car used only by the internal calendar; `Booking` belongs to a `FleetCar`.
- Booking dates are date-only, stored as UTC midnight. End date = return day (inclusive). **Same-day turnover is allowed**: conflict check is strict (`startDate < newEnd && endDate > newStart`) so a booking may start the day another ends. Calendar bars render mid-day to mid-day to visualize this. Don't "fix" this with >= comparisons.
- Admin auth: single password (`ADMIN_PASSWORD` env) â†’ HMAC-signed cookie (`lib/auth.ts`). The admin layout renders the login form when unauthenticated; every server action calls `requireAdmin()`.
- Uploads: `POST /api/upload` writes to `public/uploads/` (local-only; swap for Blob storage before deploying).
- Brand: indigo `#383E7E`, green `#008067` (accent/CTA), slate `#464555`; fonts Saira (display) + Raleway (body) from `app/fonts/`. Source assets in `INDIGO CARS BRANDING zip/`. The PUBLIC site is a LIGHT theme (white bg, indigo headings, green CTAs â€” matches the business cards); the ADMIN panel uses the dark `ink`/`ink-soft` tokens. Don't make the public site dark again â€” the owner rejected that.
- Car photos in `public/cars/*.jpg` are AI **studio renders** (silver, front 3/4, on white â€” the "Budget" catalogue look) generated with Higgsfield's `marketing_studio_image` model. To add/regenerate one, generate at that style and drop the jpg in `public/cars/`. `public/cars/_coming-soon.svg` is the placeholder for models without an image yet. The fleet/data is Indigo Cars' **real** list â€” `prisma/seed.ts` (idempotent) and `scripts/reset-real-fleet.mjs` (wipe & reload) are the source of truth. Real contact info (from the business card): +90 546 646 01 53 / info@indigocars.co / www.indigocars.co.
- **Car images resolve through `lib/carImage.ts`** (`resolveCarImage`). When `IMAGIN_CUSTOMER_KEY` is set, cards render clean imagin.studio studio side-profiles (angle 21) from each car's `imaginMake`/`imaginModel` slugs; with no key (or no slugs) they fall back to the stored `car.image`. Resolution happens server-side in `app/page.tsx` (keeps the key off the client) before passing cars to `FleetSection`. Slugs are seeded (`prisma/seed.ts` IMAGIN_SLUGS / `scripts/set-imagin-slugs.mjs`) and editable per car in the admin. Confirmed angle 21 = the clean side-profile-on-white "Budget" look; unlicensed keys watermark, paid keys are clean.

- **i18n** (`lib/i18n.ts`, pure/client-safe): public site is multilingual â€” `en` (editable CMS baseline) + `tr`, `ar`, `ru`, `de`. `CONTENT[locale]` = built-in translations of the marketing text (keys in `LOCALIZED_KEYS`); `UI_STRINGS[locale]` = chrome (nav, fleet labels, category/fuel/transmission maps, bento, contact). Locale resolution is server-only in `lib/locale-server.ts` `getRequestLocale()`: `lang` cookie wins, else `pickLocale(Accept-Language)` auto-detects the browser language. Used by `app/layout.tsx` (`<html lang dir>`, `ar`=rtl) and `app/page.tsx`. Switch via `components/LangSwitcher.tsx` (sets cookie + reload). Admin forced `dir="ltr"`.
- **Editable translations**: `getLocalizedSettings(locale)` (lib/settings.ts) merges defaults â†’ DB plain keys (English + shared neutral fields like phone/images) â†’ `CONTENT[locale]` â†’ DB `"<locale>:key"` admin overrides. The admin content editor (`app/admin/content`) has language tabs (`?lang=xx`); non-English tabs edit only `LOCALIZED_KEYS`, saved namespaced as `"<locale>:key"` by `saveSettings` (reads the `locale` form field). Shared fields (phone/email/images/links/stat numbers) are English-tab only. Car make/model names stay Latin; categories/fuel/transmission translate via `t` maps (filtering keys on canonical English category).
